This repository contains a complete series of four incremental homework assignments for the **Modern Methods of Software Engineering** course (Academic Year 2024/2025). The portfolio tracks the full lifecycle of a software project, from initial business requirements to formal object-oriented design and architectural mapping.

### Homework 1: Foundation of UML
**Focus:** Initial Modeling & Workflow Visualization
* Introduction to **Use Case Diagrams** and **Activity Diagrams**.
* Modeling business workflows for salary management and hospital resource ordering.
* Handling parallel flows and exception handling in procedural logic.

### Homework 2: Requirements Elicitation
**Focus:** The "What" of the System
* Development of a **Project Glossary** and Actor identification.
* Drafting detailed **Scenarios** to capture real-world usage.
* Formal **Use Case Descriptions** (Happy Paths, Alternatives, and Exceptions) using *include* and *extend* relationships.
* Definition of **Non-Functional Requirements** (Performance, Security, Reliability).

### Homework 3: Requirements Analysis
**Focus:** The "How" (Internal Logic)
* Application of the **Entity-Boundary-Control (EBC)** pattern to identify system objects.
* **Sequence Diagrams**: Mapping use cases to object interactions to visualize message flows.
* **Class Diagrams**: Defining the structural domain model with proper associations and multiplicities.
* **Statechart Diagrams**: Modeling the lifecycle and state transitions of critical system entities.

### Homework 4: System & Object Design
**Focus:** Architectural Implementation
* **Subsystem Decomposition**: Breaking the system into manageable, cohesive components.
* **Deployment Mapping**: Visualizing the hardware/software infrastructure using **Deployment Diagrams**.
* **Access Control**: Designing security via an **Access Matrix**.
* **Design Patterns**: Applying industry-standard patterns to solve structural and behavioral challenges.
* **Formal Contracts**: Using **Object Constraint Language (OCL)** to define rigid class invariants and pre/post-conditions.
